# Code Of Conduct

This project should be useful, direct, and welcoming.

## Expected Behavior

- Be respectful and constructive.
- Focus criticism on code, docs, and security reasoning.
- Assume good intent while still being precise about risk.
- Help new contributors understand the project before rejecting an idea.

## Unacceptable Behavior

- Harassment, insults, threats, or discriminatory language.
- Publishing private security reports before maintainers have time to respond.
- Spamming issues, pull requests, or discussions.

## Enforcement

Maintainers may edit, hide, or remove comments and may block users who repeatedly disrupt the project. Security reports should follow `SECURITY.md`.
